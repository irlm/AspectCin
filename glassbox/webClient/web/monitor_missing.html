Glassbox Monitor jar either not installed or not on the system classpath


Troubleshooting your installation....
  • Did you setup Glassbox on your Application Server? (Tomcat, WebSphere, etc)
  • Did you restart your application server?
  • Check to make sure the environment variables defined by the launching script are not overridden and are passed through when starting your Java VM. If a classpath environment variable is replaced or not used when launching Java, it can cause this problem.
  • Check your Application Server's home directory to verify the glassboxMonitor.jar file has been installed
  • Add the glassboxMonitor.jar file as the first entry in your application server's system classpath. Consult your server documentation for more info
  • Check on the Glassbox Forums
  • Contact Glassbox Support
